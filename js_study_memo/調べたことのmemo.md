# TypeScript

1. TypeScript は単に JS の拡張的ポジションで method とか追加されるもんだと思ってた。
   1. 実際のところは、TypeScript をコンパイルして js ファイルを生成するぐらいだし、スパーセットとして包括してるイメージのほうが近い。
   2. だからと言って、js でできないことは TypeScript でもできない。あくまで js でできることを簡潔に書いて開発できるようにしてるだけ。元に interface のソースとかはコンパイルしても js ファイル側にソースコードは作られない。
2. 色々調べて感じたけど、Rails で学んだことがそのまま TypeScript でもできるイメージ。
   1. get,set しかり、private,public とか継承・overwrite とか js でもできることとの区別をちゃんとしないとだね。
3. TypeScript の良さはもちろんわかるが、書くべきことが多すぎて少しだるい気もする。背景のイメージとして大規模開発があるからしょうがないけど、進みたい速度で最初進めない気がする。最終的には急げば回れではないが、この保守性が速度を上げる根拠にもなりそうだけれども。。。。Rspec を書いてる時の姿勢に近い。
